# Code Analysis

用于先理解代码，再给结论。目标不是复述文件内容，而是回答“这段代码到底在做什么、影响什么、问题在哪、该关注哪里”。

## 工作方式

1. 先确认分析目标。
   明确是要看职责、调用链、数据流、影响面、异常原因，还是架构结论。
2. 只读必要文件。
   优先读入口、调用方、被调方、配置和测试，避免无边界扫全仓。
3. 区分事实和推断。
   每个结论都尽量绑定到文件证据；无法确认时明确写成推断。
4. 输出结论。
   默认按 [references/code-analysis-template.md](references/code-analysis-template.md) 的结构组织结果。

## 输出要求

- 先写结论，再写证据
- 明确影响范围和风险点
- 如果有未证实部分，标记为 `待确认`

## 给维护者的提示

- 这个 skill 只负责分析，不负责写方案或直接改代码
- 如果团队更看重某类输出，优先调整 `references/` 模板
