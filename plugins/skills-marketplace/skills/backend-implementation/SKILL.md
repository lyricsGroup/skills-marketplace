# Backend Implementation

用于后端编码落地。重点是把接口契约、业务逻辑、数据变更和验证动作收敛清楚。

## 工作方式

1. 明确契约。
   先确认输入、输出、异常、权限、幂等性和兼容性。
2. 收敛改动范围。
   识别控制器、服务、数据层、异步任务、配置和迁移脚本。
3. 完成实现并验证。
   至少检查正常路径、失败路径、边界条件和回归风险。
4. 输出结果。
   默认按 [references/backend-checklist.md](references/backend-checklist.md) 汇报。

## 输出要求

- 写明接口或数据层变更
- 写明迁移、配置或兼容性影响
- 写明测试或验证覆盖

## 给维护者的提示

- 项目级约束优先写到 `references/`
- 涉及数据库或任务流时，保持检查项显式可见
