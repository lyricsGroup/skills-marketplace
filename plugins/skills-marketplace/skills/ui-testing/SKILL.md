# UI Testing

用于在实现完成后检查 UI 质量。重点是关键路径是否可用，视觉和交互是否存在明显缺陷。

## 工作方式

1. 确认测试范围。
   明确页面、设备、浏览器、关键流程和重点风险。
2. 按流程检查。
   优先测主流程，再测异常路径、边界状态和视觉一致性。
3. 分类问题。
   将问题区分为阻塞、一般缺陷、建议优化。
4. 输出结果。
   默认按 [references/ui-test-checklist.md](references/ui-test-checklist.md) 返回通过情况和问题列表。

## 输出要求

- 写明测试范围和通过情况
- 写明复现步骤和实际表现
- 给出整体 UI 结论

## 给维护者的提示

- 如果团队使用固定验收标准，把标准写进 `references/`
- 这个 skill 输出的是测试结论，不替代代码审核
