# Performance And Resilience Guide

## 熔断与降级

- 对关键外部依赖考虑熔断，避免单点故障拖垮整体链路
- 降级策略要明确：返回兜底值、局部失败、缓存数据或明确提示不可用
- 重试不是默认正确答案，要结合幂等性、超时和流量放大风险判断

## 性能

- 先识别核心热点接口，再决定优化优先级
- 关注慢查询、全表扫描、大对象序列化和无边界分页
- 不要为了“代码优雅”牺牲链路开销可控性

## 数据库查询

- 设计查询前先想清楚 where、order by、limit 和索引
- 查询出来的数据应服务明确用途，不要无意义多取字段
- 对统计和搜索类接口，优先评估成本而不是先写出来再说

## 发布前检查

- 慢路径是否可观测
- 依赖故障是否会级联
- 高并发下是否有明显瓶颈
- 查询和缓存策略是否能支撑预期流量
