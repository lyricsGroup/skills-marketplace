# API Contract Guide

## 输入输出

- 请求参数命名清晰且稳定
- 返回结构表达真实业务语义
- 可选字段、默认值、分页和排序规则显式说明

## 参数校验与 400 防护

- 对必填项、格式、长度、范围、枚举值做前置校验
- 错误信息要让调用方知道哪里错了，而不是只返回模糊 bad request
- 不要把本可在边界层拦住的错误放到服务层或数据库层才暴露
- 参数校验规则要稳定，避免同类接口行为不一致

## 错误语义

- 区分参数错误、权限错误、资源不存在、状态冲突和系统异常
- 调用方应该能根据返回结果采取稳定处理策略
- 不要把所有错误都折叠成一个模糊失败

## 兼容性

- 新增字段优先向后兼容
- 删除字段、修改语义、改变默认行为前要明确影响范围
- 外部接口改动必须考虑已有调用方和历史客户端
