# Frontend Implementation

用于前端编码落地。前提是需求或方案已经足够明确；如果不明确，先回到 `solution-planning` 或至少收敛假设。

## 工作方式

1. 确认边界。
   明确页面、组件、状态、接口、空态、错误态和响应式范围。
2. 遵循现有设计系统。
   如果仓库已有组件库、样式约定或视觉语言，优先沿用，不另起体系。
3. 完成实现并校验。
   至少检查加载态、空态、报错态、交互反馈和移动端基本可用性。
4. 输出结果。
   默认按 [references/frontend-checklist.md](references/frontend-checklist.md) 自检。

## 输出要求

- 说明改了哪些页面或组件
- 说明涉及哪些状态和边界情况
- 如果无法完整验证，明确留下验证缺口

## 给维护者的提示

- 这个 skill 聚焦前端落地，不负责完整测试报告或代码审核
- 把团队 UI 规范沉淀到 `references/`，不要堆在正文
