# React Guide

## 使用原则

- 优先函数组件和现代 React 写法
- 将副作用控制在必要边界，不让组件在每次渲染时产生不可控行为
- 组件层尽量保持声明式，复杂编排逻辑下沉到 hooks 或上层容器

## 状态与副作用

- 区分本地 UI 状态和服务端状态
- 避免把请求、派生计算、表单逻辑和展示逻辑全部塞进一个组件
- 状态提升要有充分理由，不要因为“方便传参”把所有状态抬到顶层

## 组件设计

- Props 要小而清晰
- 列表项、表单块、面板区域等可独立理解的部分应拆分
- 不要为了复用而过早抽象成难以阅读的“万能组件”

## 常见问题

- effect 里混入过多职责
- 事件回调直接串联过长业务流程
- 组件中既做布局又做数据 orchestration 又做校验
- 类型存在但没有真正约束数据边界
