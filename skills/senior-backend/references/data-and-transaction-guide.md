# Data And Transaction Guide

## 数据变更

- 任何表结构调整都要考虑历史数据
- 新字段的默认值、回填策略和空值语义要明确
- 删除或重命名字段前要确认读写链路和回滚路径

## 查询能力

- 明确每个核心查询的筛选条件、排序方式和分页策略
- 优先让查询命中索引，而不是依赖运行时侥幸
- 谨慎处理多表关联、大范围模糊搜索和聚合统计
- 识别并避免 N+1、重复查询和超大结果集

## 事务边界

- 明确哪些操作必须原子化
- 跨资源写入时考虑最终一致性方案
- 不要让事务范围过大，影响吞吐和锁竞争

## 幂等性

- 重试可能发生时，要设计幂等键或去重逻辑
- 对外部消息、异步任务、支付、回调等链路尤其重要

## 发布与回滚

- 迁移脚本要可审查
- 变更需要明确先后顺序
- 回滚不是口头承诺，要有具体路径
